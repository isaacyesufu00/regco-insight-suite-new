BUILD: Universal CBS File Parser — Accept Any Raw Bank Excel File
This is the most critical feature for RegCo's commercial viability. Right now RegCo only accepts a specific 3-sheet template. Real clients will upload raw exports from FlexCube, Ncube, Finacle, Temenos, or plain Excel GL reports with hundreds of rows. RegCo must intelligently parse ALL of these and extract the figures needed to generate CBN returns.
Build a complete universal parser that sits inside the process-report edge function and handles any file a Nigerian MFB throws at it.

PART 1 — MULTI-STRATEGY FILE DETECTION
When a file is uploaded, before any parsing, run a detection phase to identify what kind of file it is. Check these in order:
Strategy A — RegCo Template Detection (existing flow)
Check if the workbook contains a sheet named exactly "RegCo Upload Format". If yes, use the existing template parser. No changes needed for this path.
Strategy B — GL Trial Balance Detection
Check if any sheet contains BOTH a debit column AND a credit column AND more than 20 rows of data. If yes, treat it as a raw GL trial balance and use the GL parser (Part 2 below).
Strategy C — Single Sheet Summary Detection
Check if any sheet has rows where column A contains keywords like "total assets", "deposits", "capital", "loans". If yes, treat it as a financial summary sheet and use the keyword extractor (Part 3 below).
Strategy D — Multi-Sheet Workbook
If the file has 3 or more sheets, scan ALL sheets for financial data and merge findings across sheets.
After detection, log which strategy was used: console.log('Parser strategy:', strategyUsed).

For each row, check if the normalized label STARTS WITH or CONTAINS any keyword (check starts-with first for accuracy). When matched, extract the numeric value from the first non-empty numeric column to the right of the label.
Number extraction rules — handle all common formats:
javascriptfunction extractNumber(raw) {
  if (!raw && raw !== 0) return null;
  const str = String(raw).replace(/[₦$,\s]/g, ''); // remove currency symbols and commas
  // Handle thousands notation (e.g. "2,847,700" in a ₦'000 sheet = 2,847,700,000)
  const num = parseFloat(str);
  return isNaN(num) ? null : num;
}
IMPORTANT — Thousands multiplier detection:
Scan the sheet header rows for any text containing "'000" or "thousands" or "N'000" or "₦'000". If found, set multiplerIs1000 = true and multiply ALL extracted figures by 1000 before using them. Many CBN reports are submitted in ₦'000 notation.

PART 4 — INSTITUTION DETAILS AUTO-DETECTION
Search ALL sheets for institution information. Scan every cell in the first 30 rows of every sheet for these patterns:

Institution name: Any cell containing "MICROFINANCE BANK", "MFB", "LIMITED", "LTD" — take the full cell value
License number: Any cell matching pattern MFB/[A-Z]+/[A-Z]/[0-9]+/[0-9]+
Reporting period: Any cell containing month names (JANUARY through DECEMBER) followed by a 4-digit year, OR any date value in Excel date format
RC Number: Any cell where the label nearby says "RC" or "RC NUMBER" or "CAC" followed by digits

If institution name is not found in the file, fall back to the value stored in the user's profiles table (company_name field) — fetch it from Supabase using the user's ID.
